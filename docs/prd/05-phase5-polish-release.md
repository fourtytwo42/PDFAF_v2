## HTML Report Generation

When `POST /v1/remediate` is called, the response includes an optional HTML accessibility report.

```typescript
export interface HtmlReportOptions {
  includeBeforeAfter: boolean
  includeFindingsDetail: boolean
  includeAppliedTools: boolean
}

export function generateHtmlReport(
  before: AnalysisResult,
  after: AnalysisResult | null,
  appliedTools: AppliedTool[],
  options?: HtmlReportOptions
): string
```

**Report sections:**

1. **Header** — Document title, filename, report date
2. **Grade Summary** — Large letter grade, score, before/after comparison bar
3. **Category Scores** — Table with all 11 categories: score, grade, findings count
4. **Key Findings** — Bulleted list of blocking findings with WCAG criterion references
5. **Applied Repairs** — What the system fixed (only if `includeAppliedTools: true`)
6. **Remaining Issues** — What still needs manual attention
7. **Footer** — Generated by PDFAF v2, timestamp

**Design constraints:**
- Self-contained HTML (no external CDN dependencies)
- Inline CSS only (no separate stylesheet)
- < 50KB for typical reports
- Printable (CSS @media print)
- Accessible (the irony of an inaccessible accessibility report is not lost on us)

```yaml
openapi: 3.1.0
info:

    Grade and remediate PDFs for WCAG 2.1 AA / ADA Title II compliance.

    ```bash
    curl -X POST http://localhost:6200/v1/analyze \
      -F "file=@document.pdf" \
      | jq .grade
    ```
  contact:
    name: ICJIA
    url: https://icjia.illinois.gov

servers:
  - url: http://localhost:6200
    description: Local development

paths:
  /v1/analyze:
    post:
      summary: Grade a PDF
      description: Analyze a PDF and return WCAG accessibility grade without modifying it.
      requestBody:
        required: true
        content:
          multipart/form-data:
            schema:
              type: object
              required: [file]
              properties:
                file:
                  type: string
                  format: binary
      responses:
        '200':
          description: Analysis complete
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/AnalysisResult'
        '400':
          $ref: '#/components/responses/BadRequest'
        '422':
          $ref: '#/components/responses/InvalidPdf'
        '503':
          $ref: '#/components/responses/Overloaded'

    post:
      summary: Remediate a PDF
      description: |
        Analyze, repair, and re-grade a PDF. Returns the remediated PDF and a

        '200':

                $ref: '#/components/schemas/RemediationResult'

          $ref: '#/components/responses/BadRequest'

## Production Hardening

### Rate Limiting

```typescript
import rateLimit from 'express-rate-limit'

const analyzeRateLimit = rateLimit({
  windowMs: 60 * 1000,      // 1 minute
  max: 30,                   // 30 requests per IP per minute
  message: { error: 'Too many requests', retryAfter: 60 },
  standardHeaders: true,
})

const remediateRateLimit = rateLimit({
  windowMs: 60 * 1000,
  max: 10,                   // remediation is heavier
  message: { error: 'Too many requests', retryAfter: 60 },
})
```

### Request Timeout Middleware

```typescript
// Abort hung analysis after 120s
app.use('/v1/analyze', (req, res, next) => {
  const controller = new AbortController()
  req.signal = controller.signal
  const timer = setTimeout(() => controller.abort(), 120_000)
  res.on('finish', () => clearTimeout(timer))
  next()
})
```

### Structured Logging

```typescript
// Replace console.log with structured JSON logs
interface LogEntry {
  level: 'info' | 'warn' | 'error'
  message: string
  requestId?: string
  filename?: string
  analysisMs?: number
  score?: number
  grade?: string
  error?: string
  stack?: string
}

function log(entry: LogEntry): void {
  process.stdout.write(JSON.stringify({ ...entry, timestamp: new Date().toISOString() }) + '\n')
}
```

### Error Response Shape

All errors use a consistent envelope:

```typescript
interface ErrorResponse {
  error: string           // human-readable message
  code: string            // machine-readable code (e.g. "INVALID_PDF", "FILE_TOO_LARGE")
  details?: unknown       // optional debug info (omitted in production)
  requestId?: string
}
```

## Dockerfile

```dockerfile
FROM node:22-slim AS base

# Install system dependencies
RUN apt-get update && apt-get install -y \
    qpdf \
    python3 \
    python3-pip \
    tesseract-ocr \
    && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
RUN pip3 install pikepdf fonttools --break-system-packages

WORKDIR /app

# Install node dependencies
COPY package.json pnpm-lock.yaml ./
RUN npm install -g pnpm && pnpm install --frozen-lockfile

# Build TypeScript
COPY . .
RUN pnpm build

# Runtime
EXPOSE 6200
ENV NODE_ENV=production
ENV PORT=6200
CMD ["node", "dist/index.js"]
```
